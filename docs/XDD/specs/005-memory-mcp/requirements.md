# PRD — Memory + MCP Integration (M5)

## Summary

Connects the M2 file-based memory system with the M4 Satori context server. Adds skill-level
Satori detection, routes session-scoped data to Satori when available, and integrates
the Kairn handler once it ships. File memory remains the source of truth and primary system;
Satori is an enhancement layer with full graceful degradation.

---

## Prerequisites

- M2 complete ✅ (file-based memory system, hooks, routing skills)
- M4 complete — `miyo-satori` submodule installed, `satori_context` tool available

  route project-knowledge queries to Kairn instead of local SQLite; no TCS code change needed

## Routing Table Decision (Open — R6.4)

The M2 routing table maps lifetimes to file locations. M5 must decide what moves to Satori:

| Lifetime | M2 Location | M5 Candidate | Notes |
|---|---|---|---|
| static longlived | `~/.claude/includes/` | unchanged | |
| longlived | `~/.claude/includes/` | unchanged | |
| medium | `docs/ai/memory/*.md` | unchanged | |
| short | `docs/ai/memory/context.md` | unchanged or Satori | unclear; context.md is human-readable and git-tracked, losing that has cost |
| **really short** | context window | **Satori DB** | session state, task progress, recent errors — clear fit |

**Open question**: should "short lived" (context.md) also move to Satori, or stay in files?
- Files: human-readable, git-tracked, works without Satori
- Satori: faster to query, cross-session recall via FTS5, less file noise

Decision to be made during M5 spec.

## Graceful Degradation

All M5 behavior must degrade cleanly when Satori is not installed:

| Feature | Satori present | Satori absent |
|---------|---------------|---------------|
| Session restore | `satori_context restore` | Load from `docs/ai/memory/context.md` |
| Context search | `satori_context query` | Grep over memory files |
| Session guide | Generated by Satori on compaction | Manual `/memory-add` prompt |
| Kairn queries | Routed via Kairn handler | File memory only |

## Changed Behavior vs M2

| Behavior | M2 | M5 (with Satori) |

| Semantic search | Not available | `satori_context query` |
